# Prototype sources — Build hub modules

Extracted verbatim from the uploaded prototype
(`FundExecs_OS_Reboot_standalone.html`, embedded asset store). These are the
UX/UI specs the four Build-tab agents replicate — reference only, never
imported by the app.

| File | Prototype module |
| ---- | ---------------- |
| `formation.jsx.txt` | Fund Formation — the copiloted document sequence |
| `governance.jsx.txt` | Structure & governance — bodies + policy builders |
| `data-room.jsx.txt` | Materials & data room — builders, folders, links |
| `brand.jsx.txt` | Your profile & brand — kit, bio, credentials |
| `data-layer.jsx.txt` | Shared seeds, options and copy (team, mandate, workspace) |
| `ui-kit.jsx.txt` | The prototype's visual primitives (map to `components/ui/*`) |

Honest-data rule: seeded rosters/counts/activity in these files are mock
demo data. The live app surfaces real Supabase rows or honest empty states —
see `docs/agents/BUILD_TABS_PLAYBOOK.md`.
